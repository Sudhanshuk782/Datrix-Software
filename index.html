What is TarkaMo?

TarkaMo is a MATLAB/Simulink-class Model-Based Design platform built for embedded engineers who work under real safety constraints. Like Simulink, you design your system as a graphical model — state machines, block diagrams, flow graphs — then simulate it and generate production-ready Embedded C code automatically. Unlike Simulink, TarkaMo has MISRA-C, ISO 26262, and DO-178C compliance checks built directly into the simulation gate, not as expensive add-on toolboxes.

TarkaMo runs on Windows as a standalone desktop application. No MATLAB licence required. No toolchain subscription. No cloud. Every feature works offline.

Core Features
🧱
EventFlow — State Machine Editor
Drag-and-drop FSM canvas. States with IN / RUN / OUT code blocks. Transitions with guards, conditions, and actions. 5 px drag threshold prevents accidental moves. Rubber-band multi-select for copy, screenshot, comment-out, or delete.
🔀
Advanced Block Library
Joint Execution, Single-Level History, Multi-Level History, Node, SplitFlow, and MergeFlow — all accessible from the EventFlow toolbar. Logically grouped with separators matching UML/SysML semantics.
🛡
MISRA-C 2012 Validation
R5.1 identifiers ≤31 chars · R5.2 no C keywords · R5.3 valid C identifier · R8.7 state internal · R13.5 no ++ in conditions · R14.4 bare-variable boolean · R15.5 cyclomatic complexity check. All gated — failed checks block simulation.
🏭
ISO 26262 Part 6 Checks
P6T1 explicit dtype · P6C7 output initialisation · P6C9 reachable states must have exit · P6C9 non-deterministic transitions blocked. Flags as [ISO-26262] warnings distinct from MISRA-C errors.
✈
DO-178C / Avionics Checks
S6.3.1(b) all states must have code · S6.3.2 no magic numbers · S6.4.2 MC/DC flagged when >3 operators · S6.4.4 no TODO/FIXME markers. Separate flag — can be enabled independently of MISRA / ISO.
⚙
Embedded C Code Generation
Generates MISRA-C compliant .c and .h files from your model. RTOS task bindings for FreeRTOS. CAN frame map output. TRUE/FALSE normalised to Python booleans. Semicolons optional in code blocks — stripped automatically.
📐
Signal Routing — Clean Branch Lanes
Each branch from the same port gets a unique lane offset (12 px per branch index). Midpoint changed from 50% to 40% so trunk turns earlier and the destination side stays uncluttered. Junction dots offset by the same LANE_STEP.
📖
Built-in Help — 50+ Examples
Fully resizable / maximisable help dialog. Dark-themed code viewers with correct context menus (Copy / Select All in navy-white). 50 Examples tab and Temporal Logic reference tab. All code panels use a consistent styled editor.
Validation Tags Explained
TagMeaningEffect
[SEC-EF]Security / structural model error (duplicate names, undefined variables, non-deterministic transitions)Blocks simulation
[MISRA-C Rn.n]MISRA-C 2012 rule violation in state identifiers or transition conditionsBlocks simulation (when MISRA flag enabled)
[ISO-26262 PnCn]ISO 26262 Part 6 safety requirement not metWarning — blocks simulation when ISO flag enabled
[DO-178C Sn.n]DO-178C software level guideline violationWarning — blocks simulation when DO-178 flag enabled
[ADVISORY]Style note — does not affect model correctnessAdvisory only — never blocks
